# Package Offline Regions as PMTiles from Protomaps builds and USGS 3DEP

An Offline Region is packaged as three PMTiles artifacts plus fonts and a
verified manifest. The basemap (roads, trails, water, land cover, place names)
is extracted with `pmtiles extract` from a Protomaps daily planet build, which
is ODbL OpenStreetMap data explicitly published for exactly this kind of
regional extraction — the public OpenStreetMap tile service is never
bulk-downloaded or preseeded. Terrain and contours both come from the public
domain USGS 3D Elevation Program 1/3 arc-second DEM: hillshading uses
terrarium-encoded raster-dem tiles generated by our own GDAL pipeline, and
contour lines are generated with `gdal_contour` and packaged with tippecanoe.
The browser renders the artifacts with MapLibre GL JS from origin-private file
storage, so a downloaded region works entirely in airplane mode.

We rejected preseeding raster tile servers (prohibited by OSM tile policy and
out of scope as scanned-map presentation), OpenMapTiles/MapTiler and Mapbox
downloads (license terms restrict offline packaging), and the AWS Open Data
terrarium tile set (mixed DEM sources with more complex attribution than
first-party 3DEP, and no control over quantization, which our pipeline uses to
keep terrain artifacts 4x smaller). MBTiles-plus-server was rejected in favor
of single-file PMTiles archives that can be stored and range-read directly
from origin-private file storage. See `docs/offline-region/pipeline.md` for
the full pipeline and measurements.
