# Use append-only Thread history in v1

Each Thread is a single chronological, append-only stream of immutable Captures and Enrichments. Drafts can change before commit, but corrections and extensions to committed Captures are appended as new Captures rather than rewriting history. When connectivity returns, all pending Captures in the same Thread are batched into one Enrichment using the complete history as it existed when processing began; different Threads can process independently. The Enrichment records that history point, and later Captures remain safely appended but do not retroactively change the result. We are deliberately omitting visible branching in v1 to keep capture and review simple while preventing new entries from causing earlier context to be forgotten.
