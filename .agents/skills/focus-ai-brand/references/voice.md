# Focus.AI Design System — Voice & Tone

## Core Philosophy

**"Distill the signal from the noise."**

This isn't just a tagline — it's the operating principle. Every piece of content should help the reader focus on what matters. If it doesn't add signal, cut it.

---

## Client Voice

### Characteristics

- **Confident** — We know what we're doing. No hedging, no qualifiers.
- **Professional** — Polished but not stiff. A senior colleague, not a robot.
- **Clear** — Direct communication. Jargon only when it serves the reader.
- **Warm** — Human and approachable, not cold corporate.

### What Client Sounds Like

- A trusted advisor explaining strategy
- A well-edited business publication
- A senior colleague presenting findings with authority

### What Client Doesn't Sound Like

- A salesperson pushing features
- Generic SaaS marketing copy
- Overly casual or trying too hard to be relatable

### Client Writing Guidelines

**Do:**

- Lead with value, not features
- Use specific examples and outcomes
- Address the reader's challenges directly
- Be concise — respect their time
- Use active voice

**Don't:**

- Use buzzwords without substance
- Make vague promises
- Oversell capabilities
- Use exclamation points
- Start sentences with "We're passionate about…"

---

## Labs Voice

### Characteristics

- **Curious** — "We found this interesting and dug in."
- **Wry** — Dry observations, not taking ourselves too seriously.
- **Generous** — Sharing what we learned, not gatekeeping.

### What Labs Sounds Like

- A curious colleague sharing notes over coffee
- Someone who went down the rabbit hole and came back with stories
- A Bell Labs memo from 1972, if those researchers had Twitter

### What Labs Doesn't Sound Like

- A consultant pitching services
- A thought leader building a personal brand
- An academic paper (rigorous but lifeless)
- A conference recap blog ("and then the next speaker said…")

### Labs Writing Guidelines

**Do:**

- Lead with questions and observations
- Show the rabbit holes, not just conclusions
- Credit sources and speakers fully
- Make it useful, not just impressive
- Use humor through understatement

**Don't:**

- Sell services
- Use "well actually" corrections
- Be condescending or pedantic
- Use breathless hype language
- Gatekeep knowledge

## Shared Anti-Patterns

Both brands avoid these regardless of context:

| Pattern             | Example                             | Why it's wrong                       |
| ------------------- | ----------------------------------- | ------------------------------------ |
| Corporate buzzwords | "Leverage insights to unlock value" | Empty, signals nothing               |
| Hyperbole           | "Revolutionary! Game-changing!"     | Exhausting, unearned                 |
| "Well actually"     | "While they said X, technically…"   | Pedantic, shows off                  |
| Condescending       | "For those unfamiliar with…"        | Audience knows; don't explain basics |
| Hot takes           | "Here's why everyone is wrong"      | Opinions without substance           |
| Self-promotional    | "As leaders in AI transformation…"  | Let the work speak                   |
| Passive voice       | "It was determined that…"           | Who determined it? Be direct.        |

## Example Transformations

### Client

| Don't Write                                                                              | Write Instead                                                                            |
| ---------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| "Our cutting-edge AI solutions leverage machine learning to unlock unprecedented value!" | "We connect your siloed data sources and make them accessible through natural language." |
| "Contact us to learn more about our services."                                           | "Let's discuss how this applies to your situation."                                      |
| "We're passionate about helping businesses succeed!"                                     | "We help organizations find signal in their data."                                       |
| "Our team of world-class engineers…"                                                     | "We built [specific thing] that [specific outcome]."                                     |

### Labs

| Don't Write                                                                 | Write Instead                                                                      |
| --------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| "This groundbreaking talk revolutionized our understanding!"                | "Zhang's framing of 'computer use' reframes the whole agent question."             |
| "For those unfamiliar with RAG pipelines, here's a primer…"                 | "The RAG discussion assumed chunking is solved. It isn't."                         |
| "We were thrilled to attend this prestigious event…"                        | "Two days. 25 sessions. Here's what stuck."                                        |
| "In conclusion, these insights will transform how enterprises approach AI." | "Recurring pattern: scaffolding is tech debt. Invest in model capability instead." |

## Tone Modulation

The voice stays constant; the tone adjusts to context:

| Context          | Tone adjustment                      |
| ---------------- | ------------------------------------ |
| Error messages   | Direct, helpful, never blaming       |
| Success states   | Quiet confirmation, not celebration  |
| Empty states     | Informative, suggest next action     |
| Documentation    | Clear, structured, scannable         |
| Marketing copy   | Confident, specific, outcome-focused |
| Research writing | Curious, evidence-driven, generous   |
| UI microcopy     | Concise, action-oriented, human      |

## Formatting Conventions

- **Sentence case** for headings (not Title Case, unless Labs UPPERCASE h2)
- **Oxford comma** — always
- **Em dashes** — use freely for asides (not hyphens)
- **Numbers**: Spell out one through nine, numerals for 10+
- **Dates**: "April 2026" or "4 Apr 2026" in metadata
- **Technical terms**: No quotes around standard terms. Code font for actual code.
- **Lists**: Parallel structure. Start each item the same way.
